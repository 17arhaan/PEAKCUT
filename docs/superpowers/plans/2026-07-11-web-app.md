# Web App Implementation Plan (Plan 2 of 2)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The self-serve web app for Shorts Factory: sign up → paste a YouTube URL or upload a video → watch the agent pipeline run live → review scored clips with "why this clip" evidence → download. Credits, billing webhook, account purge. Fully runnable locally with ZERO external accounts; every external service (R2, Modal, Neon, OAuth, billing checkout) arrives late behind a seam built on day one.

**Architecture:** Next.js 16 App Router (TS strict) in `web/`, local-first walking skeleton (critic-merged from two architect drafts, 2026-07-11): docker Postgres + Drizzle, Auth.js v5 JWT with dev credentials provider, Storage seam (local disk → R2), Worker seam (local subprocess running the completed Python pipeline → Modal REST). One pure `importRun` mapper consumed by both worker paths. Origin: spec §2/§3/§5/§6/§7/§8; critic verdict merged 2026-07-11.

**Tech Stack:** Next.js 16 (App Router, `proxy.ts` NOT middleware.ts), TypeScript strict, Tailwind + shadcn/ui, Drizzle ORM (node-postgres driver locally; neon-http in prod swap), Auth.js v5 (JWT sessions), Playwright, zod. Worker: the existing `worker/` Python pipeline via `uv run shorts`.

## Global Constraints

- App lives in `web/` at repo root. `npm run dev` + `docker compose up -d db` must be sufficient to run EVERYTHING through task 14 — zero external accounts.
- Next.js 16: auth guard in `web/proxy.ts` (edge-safe, JWT-only check). NEVER create middleware.ts.
- All env access through `web/lib/env.ts` (zod-validated at boot; missing var errors name the var AND where to obtain it).
- run.json contract is FROZEN (source: `worker/src/shorts/pipeline.py` `_clip_entry` + `run()` tail). Mapper asserts `version` and fails loudly on mismatch. Mapping rules (critic-corrected): `t_start/t_end ← cut.t0/t1` (NOT candidate); `duration_min ← duration_processed_s/60` (NOT source.duration_s); clip status: `dropped_reason` set → 'dropped', else 'ready'; `score`/`hook`/`qa` may be null — handle. `cost_cents ← sum(agent_totals.*.cost_cents)` (LLM-only; excludes compute — comment the ceiling).
- Credits: `credit_ledger` is truth, `users.minutes_balance` is a cache updated in the same tx. Debit = atomic `UPDATE users SET minutes_balance = minutes_balance - $x WHERE id = $u AND minutes_balance >= $x RETURNING` + ledger insert, one transaction. Refunds idempotent via unique `(reason, ref)`. Webhook idempotent via unique `payments.mor_event_id`, raw-body signature verification.
- Storage seam: `putObjectUrl(key)`, `getUrl(key)`, `delete(prefix)` — LocalStorage now (web/.data/storage), R2Storage swap-in later. Worker seam: `start(job)` — LocalWorker (detached `uv run shorts`, tails agent_events.jsonl → jobs.stage, calls importRun in-process on exit) now, ModalWorker (POST trigger; heartbeats via internal HMAC route built ONLY in that swap-in task) later.
- Progress is STAGE-WEIGHTED (ingest/signals/crew/render), never clip-count (denominator unknown until Critic finishes).
- CI must pass with no secrets: Postgres service container + LocalStorage + a stub worker that copies a committed real run.json fixture. Live-service tests are env-gated skips.
- Tests: Playwright for user-visible behavior, vitest for units. Commit after every green task. NO Claude/AI attribution in any commit, file, or comment — MANDATORY.
- shadcn components only (button, card, badge, dialog, table, sonner, tabs, progress); no other UI deps. `# ponytail:`-style comments (`// ponytail:`) on deliberate ceilings.

## Tasks

### Task 1: Scaffold + landing + env
create-next-app (TS strict, App Router, Tailwind), shadcn init + components, `lib/env.ts` (zod; only DATABASE_URL + AUTH_SECRET + AUTH_DEV required in dev — every later var optional-with-gate), static landing page with pricing tiers from spec §2. Playwright config + first test (landing renders pricing). Commit: `feat(web): scaffold, env validation, landing`.

### Task 2: DB — docker Postgres + Drizzle schema
docker-compose.yml (postgres:16), full schema: users, jobs, clips, agent_events, credit_ledger (unique(reason, ref)), payments (unique mor_event_id) + Auth.js accounts/sessions/verificationTokens (tables now, adapter wired later). Columns per critic-merged trims (see plan source discussion; jobs: id/user_id/source_type/source_url/r2_key/status/stage/progress/error/duration_min/cost_cents/created_at/updated_at; clips: id/job_id/clip_index/t_start/t_end/score/hook/captions jsonb/evidence jsonb/qa jsonb/r2_key/thumb_key/status/dropped_reason). `drizzle-kit push` in dev. Unit: insert/select round-trip. Commit: `feat(web): postgres schema — spec §6 tables`.

### Task 3: Auth — Auth.js v5, JWT, dev provider, proxy.ts
`lib/auth.ts`: Credentials dev provider (AUTH_DEV=1: any email upserts user + 60-min signup grant via credits.grantSignup — stub until task 9, wire fully there), env-gated Google. JWT sessions. `web/proxy.ts` guards /dashboard, /jobs, /settings. signin page. Playwright: dev sign-in → dashboard. Commit: `feat(web): auth — dev provider, JWT sessions, proxy guard`.

### Task 4: Dashboard shell + job list
Server component: caller's jobs (status badge, cost, created), empty state, balance display. Commit: `feat(web): dashboard job list`.

### Task 5: Storage seam + local upload
`lib/storage.ts` interface + LocalStorage; `api/upload` (multipart→disk, size cap 2GB mirroring worker), `api/media/[...key]` (range-request support for video seek). Unit: round-trip. Playwright: upload small fixture. Commit: `feat(web): storage seam, local upload + serving`.

### Task 6: Worker seam + createJob
`lib/worker.ts` interface + LocalWorker (spawn detached `uv run --project ../worker shorts <source> -o <outDir>`; tail agent_events.jsonl mapping agents→stages ingest/signals/crew/render; stage-weighted progress; on exit call importRun — import wired for real in task 7, mocked here); `actions/jobs.ts` createJob (validate URL or storage key, debit stub OK-until-task-9, insert job, worker.start); dashboard/new form. Unit: createJob inserts + calls mock Worker. Commit: `feat(web): worker seam, job creation`.

### Task 7: run.json import — THE mapper
Generate the fixture FIRST: run the real worker on `worker/tests/fixtures/real_talking_head.mp4`, commit its run.json as `web/test/fixtures/run.fixture.json`. `lib/types.ts`: zod schema of the frozen contract. `lib/run-import.ts`: `importRun(jobId, runJson, keyRemap)` — pure, transactional; error{code,message} → jobs failed; clips per mapping rules (Global Constraints); agent_events.jsonl streamed → rows; cost/duration set; status=done flipped LAST. Version mismatch → loud failure. Unit tests against the committed fixture incl. null score/hook/qa and error-shape cases. Wire LocalWorker to call it. Commit: `feat(web): run.json import mapper`.

### Task 8: Live job view
`api/jobs/[id]/status` route; `jobs/[id]/page.tsx` + JobLive.tsx (2s poll, stops on terminal): stage progress bar, agent activity feed (latest agent_events, humanized: "Scout found 17 moments"), then clip grid (video player via storage.getUrl, score badge, hook, download button, dropped clips shown greyed with reason). Playwright: create job on fixture video → poll → clips appear. Commit: `feat(web): live job view — agent feed + clip grid`.

### Task 9: Credits — real
`lib/credits.ts` per Global Constraints (atomic debit, idempotent refund, grantSignup 60min). Wire: signup grant (task 3 stub), createJob debit (estimate = source duration or 30min URL default; reconcile to duration_processed on done), refund on failed + on dropped-only jobs (spec §7). Insufficient balance → friendly error on dashboard/new. Unit: no-double-spend under concurrent debits, refund idempotency. Commit: `feat(web): credits ledger — atomic debit, auto-refund`.

### Task 10: "Why this clip" evidence panel
Dialog per clip: score.total, verdict, per-component score/25 + cited claims formatted ("energy_peak @ 00:14 — +2.1σ"), candidate source (scout rule vs llm vs fallback), QA history + repairs. Pure read of clips.evidence jsonb. Playwright: dialog shows component + claim. Commit: `feat(web): evidence panel`.

### Task 11: Caption-style switcher
Style dropdown (s1/s2/s3) → server action → LocalWorker `render_style` invocation (`shorts render --from <outDir> --style sN`) → import re-keys new mp4s (run-<style>.json path). Playwright: switch → new clip variant appears. Commit: `feat(web): caption style re-render`.

### Task 12: Settings + account purge
Profile display, delete account: storage.delete(user prefix) + cascade delete rows (spec §5). Playwright: delete → signed out, rows gone. Commit: `feat(web): settings, account deletion`.

### Task 13: Billing webhook (account-free)
`api/webhooks/billing/route.ts`: RAW body read, HMAC signature verify (BILLING_WEBHOOK_SECRET), parse event, upsert payments on mor_event_id (insert-or-ignore), ledger top-up only on fresh insert. Tested entirely with self-signed payloads — no Dodo account. Checkout redirect deferred to swap-in 19. Unit: duplicate event → single ledger delta; bad signature → 401. Commit: `feat(web): idempotent billing webhook`.

### Task 14: Sweeper + full E2E + CI
`api/cron/sweep`: jobs stale >30min → failed + refund (idempotent). `e2e/happy-path.spec.ts`: signup → paste fixture path/URL → live progress → clips → evidence panel → download (uses LocalWorker on the shortest fixture, or STUB_WORKER=1 mode that copies the committed fixture run.json — CI uses stub). `.github/workflows/web-ci.yml` (extends existing ci.yml pattern): Postgres service container, no secrets, stub worker. Commit: `feat(web): sweeper, e2e happy path, ci`.
